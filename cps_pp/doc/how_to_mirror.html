Currently the main CVS repository for the cps++ code is at Edinburgh on the trumpton linux box. The Columbia group wanted a copy on their machines. Also we needed to be able to backup the CVS repository.

The rsync command can be used to back up a directory. 

setenv RSYNC_RSH ssh
mkdir CVS
rsync -avz trumpton.ph.ed.ac.uk:/Disk/ukqcd/QCDOC/CVS/cps++  CVS
rsync -avz trumpton.ph.ed.ac.uk:/Disk/ukqcd/QCDOC/CVS/CVSROOT  CVS

To check the code out of the mirrored CVS directory: 

 cvs -d/home/mcneile/mirror/CVS/ co cps++
where /home/mcneile/mirror/CVS/ should be replaced by the full path length of your mirrored CVS directory.

Please don't use this system to make changes to the mirrored CVS directory. This system is purely for backup. There are systems that allow multiple CVS repositories, but they are more complicated.

Other options to mirror cps++

There are some more powerful ways of maintaining multiple CVS directories. The cvsup utility creates a copy of CVS repository. Only one copy can be used to easily check in code. The DCVS does allow bidirectional updating of multiple repositories. DCVS does not look easy to install. There is also a standard unix utility called rdist for copying files to many machines.
